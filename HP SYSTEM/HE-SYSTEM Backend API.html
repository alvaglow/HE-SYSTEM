Authentication & Roles
JWT-based auth with role-based access control (RBAC)

JWT Flow

Short-lived access tokens (15m) + refresh tokens (7d) stored in HTTP-only cookies. Refresh tokens rotated on every use.

Role Permissions

RoleAccess Level
studentOwn records only
teacherAssigned classes + KPI
adminAll students + invoices + staff
managementFull read + analytics
partnerOwn referrals + commissions
parentLinked children records

Core Table: students
CREATE TABLE students (
  id              UUID          PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id         UUID          REFERENCES users(id) ON DELETE CASCADE,
  student_id      VARCHAR(20)   UNIQUE NOT NULL,    -- e.g. APD21110001
  programme_id    UUID          REFERENCES programmes(id),
  intake_date     DATE          NOT NULL,
  semester        SMALLINT      DEFAULT 1,
  status          VARCHAR(20)   DEFAULT 'active',  -- active|inactive|graduated|suspended
  class_mode      VARCHAR(10)   DEFAULT 'hybrid',  -- hybrid|campus|remote
  referring_partner_id UUID     REFERENCES partners(id),
  guardian_id     UUID          REFERENCES users(id),
  created_at      TIMESTAMPTZ   DEFAULT NOW(),
  updated_at      TIMESTAMPTZ   DEFAULT NOW()
);
CREATE INDEX idx_students_status    ON students(status);
CREATE INDEX idx_students_programme ON students(programme_id);
Core Table: attendance_sessions (OTP)
CREATE TABLE attendance_sessions (
  id          UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  class_id    UUID        REFERENCES classes(id),
  teacher_id  UUID        REFERENCES teachers(id),
  otp_code    VARCHAR(10) NOT NULL,
  session_date DATE       NOT NULL,
  start_time  TIME        NOT NULL,
  expires_at  TIMESTAMPTZ NOT NULL,
  is_active   BOOLEAN     DEFAULT TRUE,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);
-- OTP codes also cached in Redis with TTL = session duration
-- Key: otp:{session_id}  Value: otp_code  TTL: 900s (15min)
Core Table: kpi_reports
CREATE TABLE kpi_reports (
  id            UUID        PRIMARY KEY DEFAULT gen_random_uuid(),
  teacher_id    UUID        REFERENCES teachers(id),
  period_month  DATE        NOT NULL,                -- first day of month
  overall_score NUMERIC(5,2) NOT NULL,
  grade         CHAR(2)     NOT NULL,               -- A, A-, B+, B …
  status        VARCHAR(20) DEFAULT 'draft',        -- draft|submitted|approved
  submitted_at  TIMESTAMPTZ,
  approved_by   UUID        REFERENCES users(id),
  notes         TEXT,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(teacher_id, period_month)
);
Core Table: partner_commissions
CREATE TABLE partner_commissions (
  id              UUID           PRIMARY KEY DEFAULT gen_random_uuid(),
  partner_id      UUID           REFERENCES partners(id),
  student_id      UUID           REFERENCES students(id),
  invoice_id      UUID           REFERENCES invoices(id),
  rate            NUMERIC(5,2)  NOT NULL,    -- percentage at time of earning
  base_amount     NUMERIC(10,2) NOT NULL,
  commission_amt  NUMERIC(10,2) NOT NULL,
  status          VARCHAR(20)   DEFAULT 'pending',  -- pending|paid|cancelled
  period_month    DATE           NOT NULL,
  paid_at         TIMESTAMPTZ,
  created_at      TIMESTAMPTZ    DEFAULT NOW()
);

// Server-side: OTP session broadcast
io.to(`class:${classId}`).emit('otp:started', {
  sessionId, code, expiresAt,
  className: 'EAP301'
});

// Student client: join class room on login
socket.emit('join:class', { classId, token });

// Student marks attendance
socket.emit('attendance:submit', {
  sessionId,
  otpCode: '472819'
});

// Teacher receives live update
socket.on('attendance:marked', ({ studentId, status }) => {
  // update student list in real-time
});

// Notification broadcast to user room
io.to(`user:${userId}`).emit('notification:new', {
  id, type: 'fee', urgent: true,
  title: 'Fee Overdue', msg: '...'
});
Security Implementation
Production-grade security controls

Rate Limiting

100 req/min per IP globally. Login endpoint: 5 attempts/15min. OTP verification: 3 attempts per session before lockout. Stored in Redis.

Input Validation

All request bodies validated with Zod schemas before processing. SQL injection prevention via Prisma parameterised queries. XSS sanitisation via DOMPurify on text inputs.

RBAC Middleware

Every route has a role guard. Fine-grained ownership checks (student can only access own records). Middleware order: JWT decode → role check → ownership check.

Data Encryption

Passwords hashed with bcrypt (12 rounds). Bank account numbers encrypted at rest with AES-256-GCM. All PII fields marked and audited. HTTPS everywhere (TLS 1.3).

Audit Logging

All write operations logged to audit_logs table: who, what, when, old value, new value. Immutable — no UPDATE/DELETE on audit rows. 90-day retention.

CORS & Headers

Strict CORS whitelist (no wildcard). Helmet.js for security headers: CSP, HSTS, X-Frame-Options, Referrer-Policy. Cookie: SameSite=Strict, Secure, HttpOnly.

version: '3.9'

  api:
